# Инструкция по деплою на GitHub Pages

## Шаги для деплоя:

### 1. Подготовка репозитория

Убедитесь, что ваш репозиторий на GitHub называется правильно. Если имя репозитория отличается от `devfolio-main`, обновите `next.config.js`:

```js
basePath: process.env.NODE_ENV === 'production' ? '/ваше-имя-репозитория' : '',
assetPrefix: process.env.NODE_ENV === 'production' ? '/ваше-имя-репозитория' : '',
```

### 2. Настройка GitHub Pages

1. Перейдите в Settings вашего репозитория на GitHub
2. Выберите "Pages" в левом меню
3. В разделе "Source" выберите "GitHub Actions"
4. Сохраните изменения

### 3. Запуск деплоя

1. Убедитесь, что все изменения закоммичены:
   ```bash
   git add .
   git commit -m "Prepare for GitHub Pages deployment"
   ```

2. Запушьте изменения в ветку `main`:
   ```bash
   git push origin main
   ```

3. GitHub Actions автоматически запустит деплой

### 4. Проверка деплоя

- Перейдите в раздел "Actions" вашего репозитория
- Дождитесь завершения workflow "Deploy to GitHub Pages"
- Ваш сайт будет доступен по адресу: `https://ваш-username.github.io/ваше-имя-репозитория/`

### 5. Локальная проверка билда

Перед пушем можно проверить билд локально:

```bash
npm run build
```

Собранные файлы будут в папке `out/`

## Важные замечания:

- Если ваш репозиторий называется по-другому, обязательно обновите `basePath` и `assetPrefix` в `next.config.js`
- После каждого пуша в `main` будет автоматически запускаться деплой
- Первый деплой может занять несколько минут
